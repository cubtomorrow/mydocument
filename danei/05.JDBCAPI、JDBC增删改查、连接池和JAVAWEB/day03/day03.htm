1   改造Connection对象的close方法

1.1    需求：

将Connection对象的close方法改造，实现close方法可以还回池中而不是直接关闭。

Ø  实现方式1：继承

优点：子类继承父类后，子类用来扩展父类的功能

缺点：父类本身的功能没有被改变

Connection对象是通过三个数据库连接的参数，数据库给我们返回的代表数据库连接的对象。DriverManager.getConnection();

 

实现方式2：装饰者模式

也叫包装者模式decorator，对对象本身的功能进行扩展。

1.2      开发步骤：

1.2.1  写一个包装类ConnectionDecorator，实现与被包装类Connection相同的父类或者接口

WebServer项目

模拟服务器技术。接受请求 + 处理响应。

从地址栏输入内容，敲回车开始，这时候浏览器去访问了服务器，服务器接受到浏览器的请求后，处理需求，最终把数据返回给浏览器。

3   项目概述

4   项目结构

创建cn.tedu.core   ----- 核心代码包

                   --WebServer类  代表服务器端程序

                   --ClientHandler类  抽取代码

创建cn.tedu.http   ---  与http协议相关的代码

                      --HttpRequest对象，封装请求信息

                      --HttpResponse对象，封装响应信息

创建cn.tedu.context    --  全局的，配置参数信息

                         --ServerContext ，封装服务器的参数

                         --HttpContext，封装http协议相关的参数

创建config目录    --- 配置文件

                   -- web.xml  配置全局信息

5.1.1  问题：

在服务器向浏览器做出数据的响应时，应该遵循http协议。

http协议规定了响应的数据格式：

状态行：HTTP/1.1 200 OK

若干响应头：Server: nginx/1.4.3

Date: Tue, 10 Jul 2018 07:17:34 GMT

Content-Type: text/html; charset=gbk

Last-Modified: Tue, 10 Jul 2018 06:46:37 GMT

Transfer-Encoding: chunked

Connection: keep-alive

Vary: Accept-Encoding

Content-Encoding: gzip

空白行：分隔响应头和响应实体

实体：。。。

明明有数据，却不显示

 

5.1.2  解决方案

 

package cn.tedu.core;

 

import java.io.IOException;

import java.io.OutputStream;

import java.io.PrintStream;

import java.net.ServerSocket;

import java.net.Socket;

 

/**

 * 这个类用来代表服务器端的程序

 * 1，声明ServerSocket对象

 * 2，在构造函数中初始化ServerSocket

 * 3，创建start方法，用来接收客户端请求并响应

 * 4，创建main方法，启动服务器

 * 5，测试

 */

public class WebServer {

   //1，声明ServerSocket对象

   private ServerSocket server;

  

   //2，在构造函数中初始化ServerSocket

   public WebServer(){

      try {

         server = new ServerSocket(8080);

      } catch (IOException e) {

         e.printStackTrace();

      }

   }

  

   //3，创建start方法，用来接收客户端请求并响应

   public void start(){

      try {

         while(true){

            //接收客户端请求

            Socket socket = server.accept();

        

            //输出内容，在浏览器上无响应，不符合http格式

            /*OutputStream out =

                   socket.getOutputStream();

            out.write("hello server~~".getBytes());

            out.flush();*/

           

            //组装符合http协议的响应的数据格式

            PrintStream ps  = new PrintStream(

                            socket.getOutputStream());

            //拼装状态行，协议名+状态码+描述

            ps.println("HTTP/1.1 200 OK");

            //若干响应头，用来告诉浏览器要显示的数据格式

            ps.println("Content-Type:text/html");

            //响应的数据的长度

            String data="hello server!!!";

            ps.println("Content-Length:"+data.length());

            //空白行

            ps.println();

           

            //响应实体

            ps.write(data.getBytes());

            ps.flush();

            socket.close();

         }

      } catch (IOException e) {

         e.printStackTrace();

      }

     

   }

  

   //4，创建main方法，启动服务器

   public static void main(String[] args) {

      WebServer server = new WebServer();

      server.start();

   }

  

}

   private ExecutorService threadPool;

  

  

   //2，在构造函数中初始化ServerSocket

   public WebServer(){

      try {

         server = new ServerSocket(8080);

        

         //b,,,,初始化线程池

         threadPool =

                Executors.newFixedThreadPool(100);

        

      } catch (IOException e) {

         e.printStackTrace();

      }

   }

  

   //3，创建start方法，用来接收客户端请求并响应

   public void start(){

      try {

         while(true){

            //接收客户端请求

            Socket socket = server.accept();

           

            //c,,,,利用线程池执行线程类

            threadPool.execute(

                   new ClientHandler(socket));

         }

      } catch (IOException e) {

         e.printStackTrace();

      }

     

   }

  

   //4，创建main方法，启动服务器

   public static void main(String[] args) {

      WebServer server = new WebServer();

      server.start();

   }

  

}

5.5    动态响应文件

开发思路：

根据浏览器在地址栏输入的内容，可以从请求行中分析出来。

如：

地址栏输入：http://localhost:8080/12345.html

请求行：GET /12345.html HTTP/1.1

这时，需要获取请求行作为大的字符串，再根据空格切割字符串，返回的字符串数组，获取脚标为1的元素，动态拼接file的路径即可。
